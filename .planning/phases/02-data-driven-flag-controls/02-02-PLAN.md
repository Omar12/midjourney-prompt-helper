## Key interfaces for executor

Existing model.ts line 21: schemaVersion: z.literal(1) — change to z.literal(2).
New field to add after the flags array line: selectedVersionId: z.string().nullable().default(null)

Existing buildSession.ts clearAll (line 53): set({ intent: '', chips: [] }) — must extend to also
reset selectedVersionId: null, flagValues: {}, setFlags: {}.

The store separates flagValues and setFlags because D-06 requires that a flag set to a
default-looking value (e.g. stylize=0) still emits. isSet=true with value=0 is distinct from
isSet=false. setFlag marks isSet=true; unsetFlag marks isSet=false but leaves the value intact
(so switching back to a version restores the last-set value per hide-but-retain behavior).

Per-field Zustand selector pattern (MANDATORY — avoids referential instability):
  const flagValues = useBuildSession((s) => s.flagValues)
  const setFlags = useBuildSession((s) => s.setFlags)
  Never: useBuildSession((s) => ({ flagValues: s.flagValues, setFlags: s.setFlags }))
</context>
