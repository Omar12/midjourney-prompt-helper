## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → store (--no field) | TextFlagControl receives raw user typing; sanitize() must run before the value is committed to the store |
| User input → store (AR custom field) | ARControl receives raw text; /^\d+:\d+$/ validation must pass before setFlag is called |
| Rendered values → DOM | All flag values rendered as React controlled input values (value prop) — not as innerHTML |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | TextFlagControl --no injection (user enters "trees --ar 16:9") | mitigate | sanitize() called on blur converts "--ar" → "-ar"; stored value is "trees -ar 16:9". The structural injection is neutralized. (ASVS V5) |
| T-02-02 | Tampering | ARControl custom w:h injection (user enters "16:9 --stylize 999") | mitigate | /^\d+:\d+$/ regex validation rejects input containing spaces or -- sequences before setFlag is called; error message shown to user. (ASVS V5) |
| T-02-04 | Denial of Service | TextFlagControl overly long --no input | mitigate | maxLength={def.control.maxLength} (500 for --no flag) applied to Input element; enforced by browser. Value comes from FLAG_DEFINITIONS def.control.maxLength — not hardcoded in component. |
| T-02-xss-01 | Spoofing | XSS via flag values rendered in controlled inputs | accept | React renders Input value= prop as attribute value, not innerHTML; no XSS vector. No dangerouslySetInnerHTML used anywhere in these components. |
| T-02-05 | Tampering | Version stored as display label (Pitfall 5 from RESEARCH.md) | mitigate | VersionSelect uses version.id as SelectItem value prop; version.label is display-only; store only ever receives the id string |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new npm packages installed in this plan |
</threat_model>
