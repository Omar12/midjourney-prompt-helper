  FLAG_DEFINITIONS: FlagDefinition[] — canonical ordering for draft.flags

From src/state/buildSession.ts (Plan 02):
  selectedVersionId: string | null
  flagValues: Record<string, unknown>
  setFlags: Record<string, boolean>
  setFlag, unsetFlag, setVersion, clearAll

App.tsx draft.flags derivation algorithm (replaces flags:[] in the serialize call):
  1. Get supportedIds: if selectedVersionId is not null, find VERSION_DEFINITIONS entry and use its
     supportedFlagIds; else use FLAG_DEFINITIONS.map(f=>f.id) (all flags when no version set)
  2. Build flags array: FLAG_DEFINITIONS
       .filter(def => supportedIds.includes(def.id) && setFlags[def.id] === true)
       .map(def => ({ flagId: def.id, value: flagValues[def.id] }))
  This preserves canonical FLAG_DEFINITIONS order (ar, stylize, chaos, seed, no) for deterministic output.
  CRITICAL (Pitfall 2): the filter must check supportedIds — a flag set on V7 but not supported by
  the new version must NOT appear in the output even if setFlags[flagId] is still true.

FlagControls.tsx prop wiring for leaf components:
  SliderFlagControl props: def, value=(flagValues[def.id] as number ?? def.control.min), isSet=(setFlags[def.id]??false), onChange=(v)=>setFlag(def.id, v), onClear=()=>unsetFlag(def.id)
  NumberFlagControl props: def, value=(flagValues[def.id] as number ?? null), isSet=(setFlags[def.id]??false), onChange=(v)=>setFlag(def.id, v), onClear=()=>unsetFlag(def.id)
  TextFlagControl props: def, value=((flagValues[def.id] as string)??''), isSet=(setFlags[def.id]??false), onChange=(v)=>{ if(v) setFlag(def.id, v) else unsetFlag(def.id) }, onClear=()=>unsetFlag(def.id)
  NOTE: TextFlagControl already sanitizes and commits only on blur/Enter (Plan 04 contract) — onChange always receives an already-sanitized, committed string. FlagControls MUST NOT call sanitize() again here.

ClearDialog hasContent update (current: intent.trim()!=='' || chips.length>0):

| T-02-ver-gate | Tampering | Version-switch flag leak into serialized output (Pitfall 2) | mitigate | App.tsx flags derivation filters by both setFlags[id]===true AND supportedIds.includes(id) — a retained flag from a previous version is excluded from draft.flags even if setFlags is still true. Serializer also calls serializeFlag which returns null for unknown flagIds (defense-in-depth). |
| T-02-xss-02 | Spoofing | XSS in LivePreview via flag values | accept | LivePreview receives a string (the serialized prompt) and renders it as a React text child. No dangerouslySetInnerHTML. Verified in Phase 1 — unchanged. |
| T-02-03 | Spoofing | XSS in FlagControls rendered text | accept | Flag labels (def.label) come from FLAG_DEFINITIONS (authored constants, not user input). Flag values rendered in controlled inputs (value= prop) or as text children — no innerHTML. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this plan |
