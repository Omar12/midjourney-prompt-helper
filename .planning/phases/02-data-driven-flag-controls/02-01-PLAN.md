## Key interfaces provided to executor

From src/domain/prompt/model.ts (Zod conventions to replicate):
- Pattern: z.object({...}) exported with z.infer<typeof X> as TS type
- Pattern: z.discriminatedUnion('type', [...]) for variant control specs
- Pattern: z.unknown() for open-ended value fields
- One schema per concept; .optional() for absent fields; .default() only for safe defaults

The ControlSpec discriminated union has 4 branches: slider ({type, min, max, step}),
number ({type, min, max}), aspect-ratio ({type, presets: string[]}), text ({type, placeholder?, maxLength?}).

FlagDefinition fields: id (string), label (string), paramName (string), control (ControlSpec),
availableOn (string[] of version IDs).

VersionDefinition fields: id (string), label (string), parameter (string — the MJ flag syntax
emitted by serialize), supportedFlagIds (string[]).

FLAG_DEFINITIONS order (canonical serializer tail order): ar, stylize, chaos, seed, no.
VERSION_DEFINITIONS order: v8.1 first (latest), then v7, v6.1, niji7, niji6.

All files in src/domain/flags/ use no React or framework imports — pure TypeScript + Zod only.
Import from sibling files with relative paths (./schema, ./versions, ./definitions).
Use @/ aliases only in UI/component files, NOT in the domain layer.
</context>
