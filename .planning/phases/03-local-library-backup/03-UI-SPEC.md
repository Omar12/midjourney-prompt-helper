Inherited from Phase 1. No new tokens introduced in Phase 3.

| Token | Value | Tailwind class | Phase 3 usage |
|-------|-------|----------------|---------------|
| xs | 4px | `gap-1`, `p-1` | Icon-to-label gaps in entry rows |
| sm | 8px | `gap-2`, `p-2` | Between action buttons within an entry card |
| md | 16px | `gap-4`, `p-4` | Between entry cards; drawer inner padding |
| lg | 24px | `gap-6`, `p-6` | Drawer header padding; section separation |
| xl | 32px | `gap-8`, `p-8` | Not used in Phase 3 |
| 2xl | 48px | `gap-12` | Not used in Phase 3 |
| 3xl | 64px | `gap-16` | Not used in Phase 3 |

**Exceptions:**
- Reload and Delete buttons per entry: `size="sm"` button variant is acceptable for desktop
  at 32px height. On touch viewports, ensure row tap target spans `min-h-[44px]` via
  padding on the entry card row.
- Drawer width: fixed at `w-80` (320px) on wide viewports; full-width (`w-full`) below the
  `sm` breakpoint. This is a layout value, not a spacing token.

## Color

Inherited from Phase 1 and Phase 2 extensions. Phase 3 extends the accent reserved-for
list by one element.

| Role | CSS variable | Phase 3 usage |
|------|-------------|---------------|
| Dominant (60%) | `--background` | Drawer background surface |
| Secondary (30%) | `--card` / `--muted` | Individual entry card background |
| Accent (10%) | `--primary` | Copy button (Phase 1); active AR chip (Phase 2); enabled Switch thumb (Phase 2); Save button (Phase 3 — new) |
| Destructive | `--destructive` | Delete confirm button in AlertDialog only |
| Foreground | `--foreground` | Entry names, drawer heading, button labels |
| Muted foreground | `--muted-foreground` | Entry timestamps, empty state body, import status message, drawer subheading |
| Border | `--border` | Drawer border-left divider; entry card borders |

**Accent extended to (Phase 3):**
1. Copy button — primary CTA (inherited from Phase 1)
2. Active AR preset chip (inherited from Phase 2)
3. Enabled Switch thumb (inherited from Phase 2)
4. Save button — `variant="default"` (accent fill) in the preview pane. Save is the
   secondary CTA of this phase and must be visually distinct from the ghost Library button.

**Do NOT apply accent to:**
- Library button (use `variant="ghost"`)
- Reload prompt button per entry (use `variant="ghost"`)
- Delete prompt button per entry (use `variant="ghost"` with `text-destructive`)
- Export / Import buttons (use `variant="outline"`)
- Inline rename input focus ring (use default `--ring`)

**Destructive text treatment for Delete:**
Delete icon/button per entry uses `text-destructive` (foreground color, not background fill)
to signal danger without the visual weight of a full destructive button. The red fill appears
only inside the confirmation AlertDialog's confirm action.

## Layout Contract

### Two-Pane (unchanged, D-09 inherited)

The two-pane layout is unchanged. The library drawer overlays both panes — it is not a
third pane in the flex row.

### Preview Pane Extension (D-11)

The Save button and Library button are added to the preview pane. The preview pane now
contains:

```
Preview pane (w-full md:w-96, border-l, sticky top-0, h-screen, p-4 md:p-6, flex flex-col gap-4)
├── [header row]
│   ├── "Preview" label (h2, text-sm font-medium uppercase text-muted-foreground)
│   └── [Library →] button (variant="ghost", size="sm", icon+label, right-aligned)
├── [LivePreview]         (flex-1)
├── [CopyButton]          (variant="default" / accent fill)
└── [SaveButton]          (variant="outline") — NEW Phase 3
```

The header row is `flex items-center justify-between` to push the Library button to the
far right opposite the "Preview" label.

The Save button sits below the Copy button in the preview pane, separated by the existing
`gap-4` rhythm. Both buttons are full-width within the pane.

### Library Drawer (D-10)

A shadcn `<Sheet>` with `side="right"` slides over the entire app. It does not push or
shrink the two-pane layout — it overlays it with a backdrop scrim.

**Focal point:** The entry list (`flex-1 overflow-y-auto` area) is the primary visual
anchor of the LibraryDrawer. It occupies the dominant vertical space between the sheet
header and the pinned footer, drawing the eye immediately on open. The SheetHeader and
SheetFooter are supporting frames; the entry list is the content payload.

```
LibraryDrawer (Sheet, side="right", w-80)
├── SheetHeader
│   ├── SheetTitle: "Saved Prompts"
│   └── SheetDescription (sr-only): "Your saved prompt library. Reload, rename, or delete entries."
├── Entry list (overflow-y-auto, flex flex-col gap-3, flex-1)  ← PRIMARY FOCAL POINT
│   ├── [PromptEntryCard] × N   (when library has entries)
│   └── [EmptyState]            (when library is empty)
└── SheetFooter
    ├── [Export Library] button (variant="outline", full-width)
    ├── [Import Backup] button (variant="outline", full-width)
    └── [ImportStatus] text (caption, muted-foreground, transient — 4s then hidden)
```

The drawer uses `flex flex-col` with the entry list taking `flex-1 overflow-y-auto` to
scroll when the list exceeds the drawer height. The footer is pinned to the bottom via
`mt-auto` or `border-t pt-4`.

### PromptEntryCard Layout

Each saved entry is a card in the entry list:

```
PromptEntryCard (flex flex-col gap-1, p-3, rounded-md, bg-card border border-border)
├── [name row]
│   ├── Entry name text (Label) OR <Input> (Body) when rename is active
│   └── [inline confirm/cancel] (visible only during rename: Enter key + Escape; no explicit button)
├── [timestamp row]
│   └── Caption: "Saved Jun 27 at 14:30"
└── [action row]
    ├── [Reload prompt] button (variant="ghost", size="sm", Lucide `RotateCcw` icon + "Reload prompt")
    └── [Delete prompt] button (variant="ghost", size="sm", Lucide `Trash2` icon, text-destructive, aria-label only)
```

The action row is `flex gap-2`. Reload prompt is left-aligned; Delete prompt is right-aligned
(`justify-between`).

## Interaction Contract

### SaveButton

- Element: shadcn `<Button variant="outline">` full-width in the preview pane
- Label — idle: "Save to library"
- Label — after success: "Saved!" (flip immediately on successful Dexie put)
- Revert delay: 2000ms then back to "Save to library"
- Disabled: when builder is completely empty (no intent AND no chips AND no flags set AND
  no version selected) — same empty test as ClearButton
- On click: snapshot `buildSession` → auto-name (D-03: intent truncated to 40 chars +
  " — " + formatted timestamp) → Dexie put → "Saved!" state. No blocking dialog.
- Icon: optionally prepend Lucide `BookmarkPlus` at 16px
- Durable storage: on the first-ever save (library was empty before this write), call
  `navigator.storage.persist()` silently. Do not surface the grant/deny result to the user.

### LibraryButton

- Element: shadcn `<Button variant="ghost" size="sm">` in the preview pane header, right-aligned
- Label: "Library" with Lucide `Library` icon at 16px prepended
- On click: open the Sheet drawer

### LibraryDrawer (Sheet)

- `side="right"`, width `w-80` on `sm` and above, `w-full` on smaller viewports
- Opens with shadcn Sheet animation (slide from right)
- Closes via: X button (SheetClose), clicking the backdrop scrim, pressing Escape
- Drawer does not unmount on close — it preserves scroll position when reopened

### PromptEntryCard — Display State

- Entry name: `text-sm font-medium` (Label role), single line, truncated with `truncate`
  if it exceeds card width. Full name visible on hover via `title` attribute.
- Timestamp: `text-xs text-muted-foreground` — format: "Saved {Mon D} at {H:MM}"
  (e.g. "Saved Jun 27 at 14:30") using `createdAt` from the stored entry.
- Reload prompt button: Lucide `RotateCcw` icon (16px) + text "Reload prompt"
- Delete prompt button: Lucide `Trash2` icon (16px), `text-destructive` color, no visible
  text label (icon-only for space efficiency); aria-label: "Delete [entry name]"

### PromptEntryCard — Inline Rename

- Trigger: click the entry name text area
- Transition: name text is replaced by a controlled `<Input>` component, pre-filled with
  the current name, auto-focused, full selection on mount
- Commit: press Enter OR blur the input → update entry name in Dexie → revert to display state
- Cancel: press Escape → revert to display state without saving
- Empty name: if the user commits an empty string, fall back to the auto-generated name
  (derived from the original `intent` + timestamp stored at save time); do not allow
  a blank name to persist
- No explicit save/cancel buttons — keyboard-only interaction; this is the established
  Interaction Pattern for inline-edits in dense list UIs

### Reload Confirm Dialog (D-05 — if-dirty gate)

Triggered by clicking Reload prompt when the builder has content.

- **Dirty check:** builder is non-empty if `intent !== ''` OR `chips.length > 0` OR any
  flag is set OR `selectedVersionId` is set — same definition as ClearButton's enabled check
- **If builder is empty:** reload instantly with no dialog
- **If builder is non-empty:** open AlertDialog

AlertDialog:
- Title: "Replace current prompt?"
- Body: "Your current work will be replaced with the saved prompt. This cannot be undone."
- Cancel: "Keep current" (`<AlertDialogCancel>`)
- Confirm: "Load saved prompt" (`<AlertDialogAction>`) — styled with default (non-destructive)
  background since this is a neutral replacement, not a destructive deletion

On confirm: call the session→entry restore helper to populate all `buildSession` fields
(`intent`, `chips`, `selectedVersionId`, `flagValues`, `setFlags`) from the entry's
stored `PromptDraft`. Close the drawer after reload.

### Delete Confirmation Dialog

Triggered by clicking the Delete prompt button on an entry.

AlertDialog:
- Title: "Delete this prompt?"
- Body: ""{entry name}" will be permanently removed from your library."
  (Entry name is sanitized text already — render as React text child.)
- Cancel: "Keep it" (`<AlertDialogCancel>`)
- Confirm: "Delete prompt" (`<AlertDialogAction>`) — styled with `--destructive` background

On confirm: Dexie delete by entry `id`. List re-renders via `useLiveQuery`.

### Export Library (LIB-05)

- Element: shadcn `<Button variant="outline">` full-width in drawer footer
- Label: "Export library"
- Icon: Lucide `Download` at 16px prepended
- On click: read all Dexie entries → serialize to JSON envelope
  `{ schemaVersion: 2, exportedAt: <ISO string>, entries: [...] }` →
  trigger browser file download with filename `mj-prompt-library-YYYY-MM-DD.json`
- Disabled: when library is empty
- No loading state needed (synchronous-enough for a local text file)

### Import Backup (LIB-05)

- Element: shadcn `<Button variant="outline">` full-width in drawer footer, below Export
- Label: "Import backup"
- Icon: Lucide `Upload` at 16px prepended
- On click: open a hidden `<input type="file" accept=".json">` via programmatic `.click()`
- On file selected: read as text → JSON.parse → Zod-validate each entry individually
  (D-08) → Dexie bulk-add valid entries with fresh `crypto.randomUUID()` ids
- Do NOT show a blocking dialog before import; the merge-and-append semantics (D-07)
  make it safe to import without warning
- After import: show ImportStatus message (see below)

**ImportStatus message** (transient, inline below Import button):

| Outcome | Message |
|---------|---------|
| All valid | "Imported {n} prompt{s}." |
| Partial | "Imported {n} prompt{s}. {k} skipped (invalid format)." |
| Nothing valid | "Nothing imported — no valid entries found." |
| File read error / not JSON | "Could not read file — please try a valid JSON backup." |

Message renders in `text-xs text-muted-foreground` (Caption role). Auto-clears after 4000ms.
Render as React text child only — never `dangerouslySetInnerHTML` (D-08 security note).

### Empty State (Library is empty)

Shown in the entry list area when `useLiveQuery` returns an empty array.

```
EmptyState (flex flex-col items-center gap-3, py-12, text-center)
├── Lucide `BookOpen` icon (32px, text-muted-foreground)
├── Heading: "No saved prompts yet" (text-sm font-medium)
└── Body: "Save your current prompt with "Save to library" to start your collection."
    (text-xs text-muted-foreground)
```

No CTA button in the empty state — the Save button is already visible in the preview pane
behind the drawer.

## Security Contract

These design-level constraints extend Phase 1 and Phase 2 security requirements to Phase 3:

1. **All imported JSON is untrusted.** Parse via `JSON.parse` (catches syntax errors), then
   Zod-validate each entry individually per D-08. Never `eval()` or trust the shape of
   imported content. Treat malformed files as a silent partial-import or no-op, not a crash.

2. **Entry names rendered in the DOM are already-sanitized text.** Entry names are derived
   from `buildSession.intent` at save time, which passes through `sanitize()` before entering
   the session. Render all names as React text children — never `dangerouslySetInnerHTML`.
   This applies to: card display, delete dialog body, inline rename input value, title attributes.

3. **Import status messages contain only controlled copy + integer counts.** The message
   string is assembled from template literals with integer counts — not user-supplied file
   content. No file content is reflected into the UI.

4. **The inline rename input is controlled.** Store the renamed value via Dexie update;
   do not reflect the raw input value into any DOM context beyond the controlled `<Input>`.

5. **No telemetry.** The durable storage prompt (`navigator.storage.persist()`) result is
   consumed and discarded silently. No analytics or error reporting calls are made.

## What This Phase Does NOT Include

Explicitly out of scope — must not appear in Phase 3 implementation:

- Per-entry search, tags, folders, or sort controls — not in LIB requirements; deferred
- Update-in-place / overwrite-existing entry — deferred (D-01)

- AI palette suggestions / API key entry — Phase 4
- Tauri desktop wrapper / OS keychain — Phase 5
- Cross-device sync, cloud backup — permanently out of scope
