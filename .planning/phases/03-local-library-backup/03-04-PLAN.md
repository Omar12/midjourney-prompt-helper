<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User file selection → ExportImport component | File object from browser file picker; contents are untrusted until importLibrary validates them |
| ExportImport → importLibrary (domain) | Component passes raw File to domain function; domain function owns all parsing and validation |
| importLibrary → IndexedDB | Only Zod-validated entries with fresh ids reach Dexie (enforced in 03-03) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-15 | Tampering | ExportImport status message reflecting file content | mitigate | Status strings are template literals with integer counts (result.imported, result.skipped) and controlled copy strings only — no file content or entry names interpolated; enforced by acceptance criteria grep on status string construction |
| T-03-16 | Tampering | Hidden file input accessible without user gesture | accept | fileInputRef.current?.click() is invoked only from an explicit button onClick — no programmatic trigger without user gesture |
| T-03-17 | Tampering | Import destroying existing library (only-backup invariant, D-07) | mitigate | importLibrary uses bulkAdd with fresh UUIDs (not put/upsert); existing entries are untouched; enforced by re-import test in 03-03 |
| T-03-18 | Information Disclosure | Exported JSON file contains API keys | accept | Phase 3 stores no API keys (Phase 4 concern); exported entries contain only user-authored prompt text; file stays on user's device by browser download |
| T-03-SC | Tampering | No new npm installs in this plan | accept | No new packages; all packages pre-installed and audited in prior plans |
</threat_model>
