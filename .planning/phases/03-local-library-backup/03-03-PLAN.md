<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User file system → importLibrary | The imported .json file is fully untrusted — may be hand-edited, corrupted, or from another machine (D-09 security note) |
| importLibrary → IndexedDB | Only Zod-validated entries with fresh ids are written; raw parsed JSON is never spread directly into Dexie |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Tampering | importLibrary — malformed/garbage JSON file | mitigate | JSON.parse wrapped in try/catch returns { error: 'parse' }; no write to IndexedDB; enforced by import.test.ts "non-JSON file" test |
| T-03-10 | Tampering | importLibrary — wrong top-level shape (no entries array) | mitigate | Explicit shape guard: `!Array.isArray(raw.entries)` → return { error: 'shape' }; enforced by import.test.ts "wrong shape" test |
| T-03-11 | Tampering | importLibrary — invalid entry corrupting IndexedDB | mitigate | Per-entry LibraryEntrySchema.safeParse; only result.data (typed, Zod-validated) written to Dexie; raw parsed object never spread directly; enforced by import.test.ts "mixed entries" and "all invalid" tests |
| T-03-12 | Tampering | importLibrary — imported entries overwriting existing entries | mitigate | crypto.randomUUID() assigned to every imported entry regardless of stored id (D-07 fresh-id invariant); put() semantics would overwrite same id — bulkAdd() with generated UUIDs prevents collision; enforced by re-import test |
| T-03-13 | Information Disclosure | exportLibrary — exported file contains sensitive data | accept | Library contains only user-authored prompt text (no API keys in Phase 3); export is a user-initiated deliberate action; file stays on user's device |
| T-03-14 | Tampering | exportLibrary — eval() of imported content | mitigate | JSON.stringify only; JSON.parse (not eval) on import; `grep -c 'eval' import.ts` acceptance check enforces this |
| T-03-SC | Tampering | No new npm installs in this plan | accept | All packages (dexie, fake-indexeddb) pre-installed and [OK]-audited in 03-RESEARCH.md |
</threat_model>
