<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| buildSession → IndexedDB | Zustand state is sanitized (sanitize() at addChip entry) before it enters the session; session snapshot written to Dexie is therefore trusted content |
| Browser storage API | IndexedDB is local-only; no network transmission |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | LibraryEntry stored in Dexie | accept | Input is a snapshot of already-sanitized buildSession state; no untrusted external source in this plan |
| T-03-02 | Tampering | Entry names rendered in LibraryDrawer | mitigate | Entry names rendered as React text children only; never dangerouslySetInnerHTML; enforced in LibraryDrawer.tsx and acceptance criteria grep |
| T-03-03 | Denial of Service | Dexie put() failure | accept | Dexie put() throws on genuine IDB errors; SaveButton has no try/catch (failure surfaces as unhandled rejection — acceptable for local MVP; no data corruption possible from a failed write) |
| T-03-SC | Tampering | npm/shadcn installs | mitigate | dexie 4.4.4 + dexie-react-hooks 4.4.0 + fake-indexeddb 6.2.5 pre-installed and [OK]-audited in 03-RESEARCH.md; shadcn sheet from official ui.shadcn.com registry — no vetting gate required per 03-UI-SPEC.md Registry Safety section |
</threat_model>
