## Manual-Only Verifications

| Behavior | Requirement | Why Manual | Test Instructions |
|----------|-------------|------------|-------------------|
| Desktop app launches as native macOS window | SC#1 / PLT-01 | Requires opening a bundled `.app` — no headless assertion | Open `src-tauri/target/release/bundle/macos/*.app`; window titled "Midjourney Prompt Helper" appears |
| All features work identically on desktop | SC#2 | Cross-target behavioral parity needs a human running the flow | D-05 parity flow: intent → AI suggest → palette chips → flag controls → save → reload → copy, on BOTH web and desktop |
| Library persists across desktop app restarts | PLT-02 / SC#3 | App quit/relaunch cycle can't be unit-tested | Save prompt in `.app`, quit, reopen, confirm library entry present |
| No CSP violations in desktop webview | D-02 | Runtime CSP enforcement observed in system console | Run production `.app`; confirm no CSP errors in macOS Console and AI suggest still reaches openrouter.ai |

**Note (Pitfall 5):** web (`http://localhost:5173`) and desktop (`tauri://localhost`) are separate origins → separate IndexedDB stores. Tester must populate each target's library independently; cross-target library is intentionally not shared (JSON export/import LIB-05 is the migration path).
