<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: D-05 UAT parity verification — web and desktop targets</name>

  <what-built>
    Task 1 produced a native macOS .app bundle via `npm run tauri build`. The same React SPA that runs in the browser now runs inside a WKWebView window wrapped by Tauri. IndexedDB (Dexie `mj-prompt-library`) persists to the macOS app container. localStorage (`mj-ph-api-key`) also persists to the macOS app container. OpenRouter AI calls go through the same browser fetch path.

    Important: the web build (http://localhost:5173 or the preview server) and the desktop app (tauri://localhost) have separate IndexedDB origins — their saved prompt libraries do NOT share data. Populate each target independently during this UAT.
  </what-built>

  <how-to-verify>
    Run the full D-05 parity flow on BOTH targets. Start with the web target, then repeat on desktop.

    **Web target** — run `npm run preview` (or `npm run dev`) and open http://localhost:4173 (or http://localhost:5173):
    1. Type a plain-language intent in the input field (e.g. "cinematic foggy forest at dawn")
    2. Set at least one flag (e.g. change aspect ratio to 16:9, set stylize to 200)
    3. Click AI Suggest with a valid OpenRouter key saved in Settings — confirm palettes populate
    4. Click at least two palette chips — confirm they appear in the builder and live preview
    5. Save the current prompt to the library with a name (e.g. "UAT Web Test")
    6. Reload the page (Cmd+R) — confirm the "UAT Web Test" entry is still in the library list
    7. Click the saved entry to reload it — confirm builder state is fully restored (intent, chips, flags)
    8. Copy the prompt — confirm a formatted string with chips and flags is on the clipboard

    **Desktop target** — open `src-tauri/target/release/bundle/macos/*.app` (double-click or `open`):
    1. Confirm the window title reads "Midjourney Prompt Helper" and the window is approximately 1440×900
    2. Repeat UAT steps 1–8 from the web target above (use the desktop as a fresh, independent target)
    3. After saving a prompt (step 5), QUIT the app completely (Cmd+Q)
    4. Reopen the .app — confirm the saved entry ("UAT Desktop Test" or whatever you used) is still in the library list (cross-restart persistence — PLT-02/SC#3)
    5. Confirm AI Suggest works (OpenRouter browser fetch passes through WKWebView)
    6. Open macOS Console.app and filter for the .app bundle name — confirm no CSP violation messages during normal use (especially during AI Suggest)

    If any step fails, document the failure with the step number and what was observed. Do NOT type "approved" until all steps pass on both targets.
  </how-to-verify>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .app bundle → user disk | The unsigned macOS .app bundle is a local dev build; not code-signed; not distributed |
| WKWebView runtime → macOS system console | Runtime CSP enforcement is visible in Console.app; any violation = a security gap to fix |
| tauri://localhost origin → IndexedDB | The desktop app's IndexedDB is isolated to the tauri://localhost origin; separate from web build |
