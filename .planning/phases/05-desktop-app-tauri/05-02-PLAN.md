<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tauri.conf.json CSP → WKWebView JS runtime | The CSP object in tauri.conf.json is the production security boundary; it gates what JavaScript in the webview can connect to |
| connect-src allowlist → localStorage API key | The API key stored in localStorage is exposed to any origin in connect-src; the allowlist must be minimal |
| src-tauri/icons/ → macOS app bundle | Icons are embedded in the .app bundle; placeholder is acceptable for dev; unsigned .app is not distributed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-02-01 | Information Disclosure | connect-src (API key exfiltration via over-broad allowlist) | mitigate | Limit connect-src to exactly `ipc: http://ipc.localhost https://openrouter.ai`. No wildcard (*). Verify with `grep -q 'openrouter.ai' src-tauri/tauri.conf.json` and confirm no `*` in connect-src. |
| T-5-02-02 | Elevation of Privilege | Tauri capability over-grant | mitigate | deny-by-default: no Tauri plugins registered (no tauri-plugin-fs, -http, -stronghold). Only `default.json` capabilities generated by `tauri init`. Zero JS→Rust IPC surface. Verify `lib.rs` has no `invoke_handler` or `#[tauri::command]`. |
| T-5-02-03 | Tampering | script-src 'unsafe-inline' | accept | 'unsafe-inline' is required for Vite/React production build and Tailwind v4 inline styles; matches the existing working web build CSP. Hardening via nonces is a future v2 security pass. This is identical to the already-approved web build posture. |
| T-5-02-SC | Tampering | npm/pip/cargo installs | accept | No new npm/pip installs in this plan. ImageMagick installed via Homebrew if needed — Homebrew is a trusted package manager on macOS; imagemagick is a widely-used, well-maintained open-source package. |
</threat_model>
