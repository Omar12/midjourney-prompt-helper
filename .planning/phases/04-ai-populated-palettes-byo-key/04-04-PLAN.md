<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Suggest button click → openRouterAdapter → openrouter.ai HTTPS | Key and intent cross the origin boundary here; must go only to openrouter.ai |
| LLM response labels → PaletteOption.tsx render | Untrusted AI text enters the DOM; must be rendered as text nodes only |
| Error message text → error banner | Fixed strings only; raw provider error bodies must not appear |
| CSP meta → browser enforcement | Restricts connect-src to self + openrouter.ai; prevents exfiltration to unexpected hosts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04-01 | Tampering | PaletteOption.tsx — label rendering | mitigate | option.label rendered as React text node only; dangerouslySetInnerHTML forbidden; sanitize() applied on promotion via addPaletteChip |
| T-04-04-02 | Information Disclosure | IntentInput.tsx — suggest handler | mitigate | Key read from useKeyStorage at call time; never logged, never included in error messages; setError uses fixed message strings from mapError |
| T-04-04-03 | Elevation of Privilege | index.html — CSP absence | mitigate | Meta CSP tag added: connect-src 'self' https://openrouter.ai; prevents any XSS from exfiltrating key to an attacker-controlled endpoint |
| T-04-04-04 | Denial of Service | Suggest button double-fire | mitigate | isLoading guard at top of click handler; button disabled while isLoading; no concurrent generateObject calls possible |
| T-04-04-05 | Tampering | Error banner — message source | mitigate | Banner renders error.message (a fixed string from mapError); raw err.responseBody or err.message from the SDK is never surfaced; no dangerouslySetInnerHTML in banner |
| T-04-04-06 | Tampering | Re-suggest chip preservation (D-08) | mitigate | setPalettes replaces available options only; existing chips in buildSession are untouched; selected-label matching uses sanitize() to prevent mismatch |
| T-04-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan; all imports from already-installed libraries |
</threat_model>
