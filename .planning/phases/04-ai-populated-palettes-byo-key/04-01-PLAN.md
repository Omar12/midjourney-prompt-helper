## Phase Goal

**As an** image creator, **I want to** enter my intent and my own LLM API key to trigger AI-suggested palette options, **so that** I can assemble a complete Midjourney prompt faster than writing it by hand.

**This plan:** Build the AI domain core — the Zod palette schema with per-category graceful degradation, the PaletteAdapter extensibility interface, and the OpenRouter implementation. This is the single place where LLM calls originate (KEY-03) and where error classification happens (AI-03). Everything in this plan is pure domain logic with colocated tests: no UI, no state.

Purpose: Establish the data contracts and network boundary all subsequent plans build on. Getting .catch([]) semantics verified here prevents silent total failures downstream (RESEARCH Pitfall 5).

Output: src/domain/ai/ directory with 6 files — schema, adapter interface, OpenRouter implementation, and their tests.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → openrouter.ai HTTPS | User's API key crosses this boundary in the Authorization header; must never cross any other boundary |
| LLM response → Zod schema | AI-returned JSON is untrusted; schema validation is the first line of defense before any data enters application state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Information Disclosure | openrouter.ts generatePalettes | mitigate | Key is accepted as argument (not module-level state); never logged, never returned in error messages; error messages use fixed strings only |
| T-04-01-02 | Tampering | PaletteResponseSchema | mitigate | Zod schema validates all 6 category fields; .catch([]) on each field prevents malformed AI output from passing unvalidated into application state |
| T-04-01-03 | Information Disclosure | mapError error messages | mitigate | Error messages are fixed strings; raw error body (err.responseBody) is never surfaced to the UI; no console.log of key, intent, or raw response |
| T-04-01-04 | Tampering | openrouter.ts provider call | mitigate | No first-party proxy; createOpenRouter({apiKey: key}) calls openrouter.ai directly; KEY-03 enforced by architecture (no server route) |
| T-04-01-SC | Tampering | npm packages | accept | ai@7.0.5 and @openrouter/ai-sdk-provider@2.10.0 passed slopcheck legitimacy audit (RESEARCH.md §Package Legitimacy Audit) |
