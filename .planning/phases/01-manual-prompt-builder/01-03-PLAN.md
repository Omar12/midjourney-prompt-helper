## Phase Goal

**As an** image creator, **I want to** type a description, add style keywords, and copy a formatted Midjourney prompt to my clipboard, **so that** I can go from a vague idea to a complete, correctly-formatted prompt without memorizing Midjourney syntax.

<objective>
Implement the Zustand build session store and the first vertical UI slice: intent input → live preview updates in real time. Then extend the slice with chip add/remove. After this plan, a real user can open the app, type an image description, add style keywords, remove them, and see the prompt assembling live. Copy and Clear are added in Plan 04.

Purpose: This is the first working vertical slice of the walking skeleton — the "live build loop" that is the core value of Phase 1. It proves that the domain core (Plan 02) integrates correctly with the UI.
Output: Running app where typing intent and adding/removing chips produces a live prompt preview. 4 component test files green (IntentInput, ChipInput, ChipArea, LivePreview).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input (textarea, chip input) → Zustand store | User-typed text enters the application state at these two points |
| Zustand store → serialize() → DOM | Assembled prompt text flows to the DOM at LivePreview |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing / Tampering | LivePreview — XSS via user-typed intent/chip text rendered in DOM | mitigate | `LivePreview.tsx` renders preview as React text child (`{preview}`) inside `<pre>` — never `dangerouslySetInnerHTML`. React's default escaping prevents XSS. Verify: `grep -rn "dangerouslySetInnerHTML" src/ui/` must return zero results. |
| T-03-02 | Tampering | ChipArea — aria-label constructed from chip.label | mitigate | `chip.label` is user-supplied plain text rendered into an HTML attribute value. React renders attribute values safely (no HTML injection possible via attribute content). `sanitize()` will gate chip labels in the full wiring (Plan 04's addChip path). For Phase 1, the risk is negligible. |
| T-03-03 | Tampering | Zustand store mutation from test environment | accept | Tests use `useBuildSession.setState()` for setup — this is the Zustand-approved test API; no production code path is affected. |
| T-03-SC | Tampering | npm/shadcn component installs | accept | No new installs in this plan — all packages from Plan 01. Package audit already complete. |
</threat_model>
